Phase 01: Ignition

How to Create the App

Run these commands in your terminal to generate the folders below.

1 Initialize

npm create vite@latest my-app

Select "React" and then "JavaScript" or "TypeScript".

2 Install

cd my-app npm install

This creates the node_modules folder.

3 Launch

npm run dev

Your app is now live at localhost:5173

Phase 02: Exploration

"What do these files do?"

The Brain & Config (Root)

node_modules/

The massive library warehouse. It contains the actual code for React and every other package you download.

Rule #1: NEVER edit anything in here. It's managed by npm.

package.json

The ID Card of your project. It lists your dependencies (libraries) and script commands (like npm run dev).

.gitignore

A list for Git. It tells your computer: "Don't upload node_modules to GitHub, it's too big!"

The Visible App (src & public)

src/ (Source)

This is your home. 99% of your work happens here. This is where you write your components.

  • main.jsx: The entry point. It hooks React into HTML.
  • App.jsx: The "Main Component". The wrapper for your whole site.
  • assets/: Images, logos, and icons go here.

public/

Contains assets that are accessed directly by the browser (like the favicon).

Critical File:

index.html

The skeleton. React injects your code into this file.

The Execution Flow

1. index.html loads
2. calls main.jsx (Entry)
3. imports App.jsx (Parent)
4. App displays YOUR COMPONENTS.
